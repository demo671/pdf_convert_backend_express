====================================================================
  STEP 1: INSTALL DEPENDENCIES
====================================================================

Open terminal in backend-express folder and run:

    npm install

====================================================================
  STEP 2: SETUP ENVIRONMENT
====================================================================

1. Copy env.example.txt to .env:
   
   Windows:
   copy env.example.txt .env
   
   Linux/Mac:
   cp env.example.txt .env

2. Edit .env file and set your values:
   - Database credentials (PostgreSQL)
   - JWT_SECRET_KEY (generate a strong random string)
   - Cloudflare R2 credentials
   - OpenAI API key (optional but recommended)

====================================================================
  STEP 3: SETUP DATABASE
====================================================================

Make sure PostgreSQL is running, then create database:

    createdb pdfportal

Or using psql:

    psql -U postgres
    CREATE DATABASE pdfportal;
    \q

Tables will be created automatically when you start the server.

====================================================================
  STEP 4: BUILD & SEED DATABASE (RECOMMENDED)
====================================================================

Run the build script to setup everything:

    npm run build

This will:
- Install all dependencies
- Validate environment configuration
- Create database tables
- Seed default data:
  * Admin user: admin@admin.com / Admin123!
  * Test client: client@test.com / Client123!
  * Default PDF processing templates

‚ö†Ô∏è  IMPORTANT: Change default passwords in production!

====================================================================
  STEP 5: START THE SERVER
====================================================================

Development mode (with auto-reload):

    npm run dev

Production mode:

    npm start

Server will start on: http://localhost:5000
(or the PORT you specified in .env)

====================================================================
  STEP 6: ADMIN USER (ALREADY CREATED)
====================================================================

If you ran "npm run build", admin user is already created:

    Email: admin@admin.com
    Password: Admin123!

To create additional admin users:

Option 1: Run the script
    npm run create-admin

Option 2: Use API (after server is running)
    POST http://localhost:5000/api/admin/users
    Content-Type: application/json
    
    {
      "email": "newadmin@admin.com",
      "password": "your_secure_password",
      "name": "Admin User"
    }

====================================================================
  STEP 7: TEST THE API
====================================================================

Health check:
    GET http://localhost:5000/health

Login:
    POST http://localhost:5000/api/auth/login
    {
      "email": "admin@admin.com",
      "password": "your_password"
    }

Use the returned token in subsequent requests:
    Authorization: Bearer <your_token>
